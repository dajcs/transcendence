# Quick Plan: Validate and Fix Uncontested Resolution Payout Skip

## Objective
Validate the bug report and fix the backend resolution payout logic so markets finalized through `_finalize_uncontested_resolutions` do not close before `trigger_payout` runs.

Purpose: `trigger_payout` intentionally exits early when `Market.status == "closed"` to prevent double payout. `_finalize_uncontested_resolutions` currently sets `bet.status = "closed"` and commits before calling it, so uncontested markets can close without crediting winners.

Output: A regression test that fails before the fix, and a focused worker-task fix that preserves `trigger_payout` as the single place that closes paid markets.

## Context
@.planning/STATE.md
@AGENTS.md
@backend/app/workers/tasks/resolution.py
@backend/app/services/resolution_service.py
@backend/tests/test_resolution.py

Key existing contracts:
- `trigger_payout(db, bet_id, outcome, overturned=False)` locks the `Market`, returns `{"already_closed": True}` if `status == "closed"`, otherwise sets `status = "closed"`, `winning_side`, `closed_at`, credits BP/TP, records surplus, applies proposer penalty if needed, then returns a summary.
- `_finalize_uncontested_resolutions(db)` selects `Market.status == "proposer_resolved"` joined to `Resolution`, skips records inside the 48h window or with any `Dispute`, then should payout the resolution outcome.
- Current bug source: `_finalize_uncontested_resolutions` sets `bet.status = "closed"` and commits before calling `trigger_payout`, causing the idempotency guard to skip payout.
