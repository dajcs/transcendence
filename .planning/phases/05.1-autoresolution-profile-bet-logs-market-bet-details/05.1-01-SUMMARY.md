# Phase 05.1 Plan 01: Resolution Source Schema and Persistence Summary

**One-liner:** Pydantic model_validator enforces open-meteo resolution_source on binary markets; market_service serializes it as JSON string into Bet.resolution_source Text column.

## What Was Built

- `MarketCreate` schema gains `resolution_source: dict[str, Any] | None = None` field
- `validate_resolution_source` model_validator rejects: non-binary market types (422), wrong provider name (open_meteo vs open-meteo), missing/empty location, condition not in {rain, temperature}
- `market_service.create_market` wires `json.dumps(data.resolution_source) if data.resolution_source else None` into the `Bet` constructor
- 6 pytest tests in `test_autoresolution.py` covering acceptance, validation rejections, and regression
