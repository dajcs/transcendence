# Phase 5.1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase adds UI to EXISTING pages only — no new routes. Three targeted additions:
> 1. Market creation form — auto-resolution toggle + City/Date/Condition fields
> 2. Market detail page — participant list, aggregate stats, payout breakdown
> 3. Profile page — full point transaction ledger below stats

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no components.json, no shadcn) |
| Preset | not applicable |
| Component library | none — hand-written Tailwind components |
| Icon library | inline SVG (existing pattern; no icon lib installed) |
| Font | system-ui (Tailwind default — no custom font declared in globals.css) |

Source: codebase scan — `components.json` absent, `globals.css` contains only `@import "tailwindcss"` + dark variant.

## Spacing Scale

Declared values (multiples of 4 only — consistent with existing codebase which uses Tailwind's default scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline separators |
| sm | 8px | Between form label and input, between table cells |
| md | 16px | Section internal padding (p-4), form field spacing (space-y-4) |
| lg | 24px | Between major sections (space-y-6 on page wrapper) |
| xl | 32px | Not used in these additions |
| 2xl | 48px | Not used in these additions |
| 3xl | 64px | Not used in these additions |

Exceptions:
- Auto-resolution collapsible sub-panel uses p-2 (8px) to visually subordinate it inside the form card
- Table row height: py-2 (8px top+bottom) — compact ledger rows

Source: existing pages use `space-y-6` page wrapper, `p-4` card padding, `space-y-3`/`space-y-1` form groups.

### 1. Auto-Resolution Toggle Panel (new, inside NewMarketPage form)

Location: after the existing deadline inputs, before the market type selector.

Structure:
```
<div class="space-y-1">
  <label class="text-sm font-medium">{t("create.auto_resolution")}</label>
  <div class="flex items-center gap-2">
    <button type="button" role="switch" aria-checked={autoResolution}
      class="rounded px-3 py-1 text-sm {active: bg-blue-600 text-white, inactive: bg-gray-200 dark:bg-gray-700 text-gray-700 dark:text-gray-300}">
      {t("create.auto_resolution_toggle")}  <!-- "Enable" or "Enabled" -->
    </button>
    <span class="text-xs text-gray-500 dark:text-gray-400">{t("create.auto_resolution_note")}</span>
  </div>

  {autoResolution && (
    <div class="rounded border border-sky-200 dark:border-sky-800 bg-sky-50 dark:bg-sky-900/20 p-2 space-y-2 mt-2">
      <div class="space-y-1">
        <label class="text-sm font-medium">{t("create.city")}</label>
        <input type="text" required placeholder="Paris" ... class="w-full rounded border border-gray-300 dark:border-gray-600 px-3 py-2 text-sm bg-white dark:bg-gray-700 text-gray-900 dark:text-gray-100" />
        <p class="text-xs text-gray-500 dark:text-gray-400">{t("create.city_hint")}</p>
      </div>
      <div class="space-y-1">
        <label class="text-sm font-medium">{t("create.condition")}</label>
        <select class="w-full rounded border border-gray-300 dark:border-gray-600 px-3 py-2 text-sm bg-white dark:bg-gray-700 text-gray-900 dark:text-gray-100">
          <option value="rain">{t("create.condition_rain")}</option>
          <option value="temperature">{t("create.condition_temperature")}</option>
        </select>
      </div>
      <p class="text-xs text-sky-700 dark:text-sky-400">{t("create.auto_resolution_info")}</p>
    </div>
  )}
</div>
```

State:
- `autoResolution: boolean` — controlled by toggle button
- When false: sub-panel hidden; `resolution_source` not sent in payload
- When true: sub-panel visible; `resolution_source: { source: "open_meteo", city: string, condition: "rain"|"temperature", date: deadlineDate }` sent in payload
- Date field is derived from deadline — not a separate input; displayed as read-only text inside the panel

Interaction:
- Toggle button click flips `autoResolution`
- Enabling auto-resolution does NOT change `resolution_criteria` input — user still writes human criteria
- Condition dropdown: two options, defaulting to "rain"
- City input: required when panel is open; freetext; placeholder "e.g. Paris"

### 4. Point Transaction Ledger (new section, ProfilePage)

Location: inside the ProfilePage `Points` tab, below the profile header and stats grid. The stats grid has five boxes: red `♥` Like Points, `BP`, `TP`, `Total Bets`, and `Win Rate`. `♥` and `Total Bets` display as integers; `BP`, `TP`, and `Win Rate` display with one decimal place.

Heading: `{t("profile.transaction_ledger")}`
Section class: `rounded border border-gray-200 dark:border-gray-700 bg-white dark:bg-gray-800 p-4 space-y-3`

Column sort: clicking column header toggles asc/desc on that column. Active sort column shows `▲` or `▼` indicator (12px, inline).

Table structure:
```
<table class="w-full text-sm">
  <thead>
    <tr class="border-b border-gray-200 dark:border-gray-700 text-xs text-gray-500 dark:text-gray-400">

      <th class="pb-2 text-right cursor-pointer select-none">{t("ledger.bp_delta")} {sortIcon("bp")}</th>
      <th class="pb-2 text-right">{t("ledger.tp_delta")}</th>

        <td class="py-2 text-right font-medium {tx.bp_delta > 0 ? 'text-green-600 dark:text-green-400' : tx.bp_delta < 0 ? 'text-red-600 dark:text-red-400' : 'text-gray-400'}">

```
No third-party skeleton library — Tailwind `animate-pulse` only.

## Interaction States

### Auto-Resolution Toggle
- Default: off (autoResolution = false), sub-panel hidden
- Toggled on: sub-panel slides in (no animation needed — just conditional render with `&&`)
- City input invalid: red border `border-red-500` on blur if empty when panel is open
- Condition dropdown: shows immediately on panel open, no async

### Participant / Payout Tables
- Initial load: skeleton (3 animated rows)
- Loaded with data: table renders
- "Show more" click: appends next 20 rows (no scroll-to-top); button shows `{t("common.loading")}` while fetching
- "Show more" hidden when no more rows (total fetched >= total count from API)
- Error: `<p class="text-sm text-red-600 dark:text-red-400">{t("market.participants_error")}</p>`

### Ledger Sort
- Click column header → sort toggles asc/desc on that column
- Other columns reset to no indicator
- Re-fetches with new sort params (not client-side sort — send `sort_by` + `sort_dir` to API)
- While refetching: show skeleton (3 rows) in table body, preserve headers

Section order (top to bottom):
1. Profile header card (avatar, username, inline mission statement editor or read-only mission, settings/friend actions)

- Pressing `Accept` saves the mission, hides the button, and replaces the input with clickable mission statement text.
- Hovering the saved mission exposes the browser tooltip `Change mission`.
- Clicking the saved mission returns to inline edit mode.

## API Contract Requirements

These endpoints are new and must be created before the UI can be implemented:

### GET /api/markets/{id}/positions
Response shape:
```json
{
  "participants": [
    {
      "username": "string",
      "side": "yes|no|<choice>|<numeric>",
      "bp_staked": 1,
      "created_at": "ISO8601",
      "user_id": "uuid"
    }
  ],
  "aggregate": {
    "total_bp": 12,
    "total_participants": 5,
    "avg_bp": 2.4,
    "by_side": { "yes": 3, "no": 2 }
  },
  "total": 5
}
```
Query params: `offset=0&limit=20`

### GET /api/markets/{id}/payouts
Response shape (only meaningful for closed markets):
```json
{
  "payouts": [
    { "username": "string", "bp_won": 4, "tp_won": 1, "user_id": "uuid" }
  ],
  "total": 3
}
```
Query params: `offset=0&limit=20`

### GET /api/users/{username}/transactions
Response shape:
```json
{
  "transactions": [
    {
      "id": "uuid",
      "date": "ISO8601",
      "type": "bet_placed|bet_won|bet_lost|withdrawal|daily_bonus|kp_allocation|payout",
      "description": "string",
      "market_id": "uuid|null",
      "market_title": "string|null",
      "bp_delta": -1,
      "tp_delta": 0
    }
  ],
  "total": 42
}
```
Query params: `offset=0&limit=25&sort_by=date&sort_dir=desc`

## Implementation Rules

1. All new strings use `useT()` hook — no hardcoded English text in JSX.
2. All new strings must be added to `en.ts`, `fr.ts`, `de.ts` simultaneously.
3. Dark mode: every new className must include `dark:` variant equivalent.
4. Loading states: every new `useQuery` must handle `isLoading` and `isError`.
5. Tables are `overflow-x-auto` wrapped — mobile-safe.
6. `UserLink` component reused for all username displays — do not inline `<a>` tags.
7. `autoResolution` state resets to false when market type changes to non-binary.
8. `resolution_source` JSON is only included in POST `/api/markets` payload when `autoResolution === true`.
9. Font weight 700 (`font-bold`) is not used in this phase — use `font-semibold` (600) for headings, `font-medium` (500) for body and labels.
