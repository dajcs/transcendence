# Phase 5.1 Plan 03: Market Positions and Payouts Endpoints Summary

**One-liner:** Two public GET endpoints exposing participant list with aggregate stats and payout breakdown for the market detail page, backed by BetPosition + BpTransaction + TpTransaction queries with SQL-level pagination.

## What Was Built

**New schemas in `backend/app/schemas/market.py`:**
- `ParticipantEntry` — user_id, username, side, bp_staked, created_at
- `AggregateStats` — total_bp, total_participants, avg_bp, by_side (dict)
- `ParticipantListResponse` — participants list + aggregate + total
- `PayoutEntry` — user_id, username, bp_won, tp_won
- `PayoutListResponse` — payouts list + total

**New routes in `backend/app/api/routes/markets.py`:**
- `GET /{market_id}/positions` — active BetPosition rows JOINed with User, aggregate stats via coalesce/sum/avg/count, side counts grouped, SQL pagination
- `GET /{market_id}/payouts` — BpTransaction(reason=bet_win) grouped by user JOINed with User, TP totals merged via dict lookup, SQL pagination

**New test file `backend/tests/test_market_positions.py`:**
- test_get_positions_empty — empty aggregate on market with no bets
- test_get_payouts_empty — empty payouts list on open market
- test_positions_public — 200 without auth cookie
- test_positions_unknown_market — 404 for non-existent market UUID
- test_positions_pagination — offset/limit params accepted, response shape verified
