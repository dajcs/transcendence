# Phase 5.1 Plan 04: Frontend UI — Auto-Resolution Toggle, Participants, Payout Breakdown, Transaction Ledger Summary

Frontend additions completing Phase 5.1 deliverables: auto-resolution toggle on market creation, participant/payout sections on market detail, and transaction ledger on user profile.

## Tasks Completed

### Task 1: i18n keys + auto-resolution toggle on market creation (commit 637ce46)

Added 35 new i18n keys across en.ts, fr.ts, de.ts covering create.auto_resolution*, market.participants*, market.payout*, ledger.*, and profile.transaction_ledger.

Modified `frontend/src/app/(protected)/markets/new/page.tsx`:
- Added `autoResolution`, `autoCity`, `autoCondition` state variables
- Inserted sky-colored sub-panel (border-sky-200/sky-800) between deadline and market type
- `resolution_source` payload uses key `location` (not `city`) and provider `open-meteo` (hyphen)
- Market type buttons reset `autoResolution` to false on non-binary selection
- Panel conditionally rendered when `autoResolution === true`

Jest tests (4 passing): toggle visibility, payload key contract (`location` not `city`), provider format (`open-meteo` not `open_meteo`), absent when toggle off.

### Task 2: Participants + Payout Breakdown on market detail, Transaction Ledger on profile (commit 52469a9)

Modified `frontend/src/app/(protected)/markets/[id]/page.tsx`:
- Added `ParticipantListResponse` and `PayoutListResponse` TypeScript interfaces
- `participantsQuery` with queryKey `["market-positions", marketId, participantOffset]`
- `payoutsQuery` with queryKey `["market-payouts", marketId]` enabled when `marketQuery.data?.status === "closed"`
- `allParticipants` accumulation via useEffect for load-more pagination
- Participants section: aggregate stats grid (total BP, YES/NO count, avg stake) + scrollable table with UserLink
- Payout Breakdown section: green-bordered, shown only when `market?.status === "closed"`
- Inserted between Live Odds and Your Position (Participants), and before Place Bet (Payout Breakdown)

Modified `frontend/src/app/(protected)/profile/[username]/page.tsx`:
- Added `TransactionEntry` and `TransactionListResponse` interfaces
- `transactionsQuery` with queryKey `["user-transactions", username, txOffset, sortBy, sortDir]`
- `handleSort()` helper resets offset+accumulation on column change
- `TYPE_COLORS` map for transaction type badges
- Transaction Ledger section: sortable Date/Type/BP columns, type badge, market link, load-more pagination
- Appended after stats grid

Jest tests (3 passing): positions URL contains marketId, payouts not called for open market, positions URL pattern.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] payoutsQuery forward reference to `market` variable**
- **Found during:** Task 2 TypeScript build
- **Issue:** `enabled: !!marketId && market?.status === "closed"` referenced `market` which is declared later via `const market = marketQuery.data`
- **Fix:** Changed to `marketQuery.data?.status === "closed"` which references the query object declared earlier
- **Files modified:** frontend/src/app/(protected)/markets/[id]/page.tsx
- **Commit:** 52469a9

**2. [Rule 3 - Blocking] Plan specified Vitest but project uses Jest**
- **Found during:** Task 1 test execution
- **Issue:** Plan's test templates used `vi.fn()`, `vi.mock()`, `vi.hoisted()` — project has no vitest config; uses Jest + jest-environment-jsdom
- **Fix:** Rewrote both test files using Jest API (`jest.fn()`, `jest.mock()`, import after mock)
- **Files modified:** Both __tests__ files
- **Commit:** 637ce46, 52469a9

**3. [Rule 1 - Bug] Generic API mock caused component type error in market-detail test**
- **Found during:** Task 2 test execution
- **Issue:** Single mock response returned participants shape for all endpoints; comments query received wrong shape causing `for...of` TypeError on non-array
- **Fix:** URL-based mock handler returning appropriate response shape per endpoint type
- **Files modified:** frontend/src/app/(protected)/markets/[id]/__tests__/market-detail.test.tsx
- **Commit:** 52469a9
