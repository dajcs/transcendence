# Phase 5.1: Autoresolution, Profile Bet Logs, Market Bet Details - Context

**Gathered:** 2026-04-16
**Status:** Ready for planning

<domain>
## Phase Boundary

Complete the Open-Meteo auto-resolution flow (market creation UI + backend wiring), add a public point transaction ledger to user profiles, and enhance market detail pages with participant lists, aggregate stats, and payout breakdowns.

</domain>

<decisions>
## Implementation Decisions

### Autoresolution
- **D-01:** Complete the existing Open-Meteo auto-resolution — the backend task (`resolution.py`) exists but the market creation form has no way to configure it. The `resolution_source` JSON field on Bet model exists but is never populated.
- **D-02:** Add a toggle "Enable auto-resolution" to the market creation form. When enabled, reveal fields: City (text input), Date (auto-filled from deadline), Condition (dropdown: rain/temperature). Store as JSON in `resolution_source`.
- **D-03:** Auto-resolution is binary only for now (weather: rain yes/no). No LLM-based or multi-API resolution in this phase.

### Profile Bet History (Point Transaction Ledger)
- **D-04:** Add a full point transaction ledger below stats on user profiles — not just bets, but ALL BP/TP/KP flows: bets placed, bets won/lost, daily login bonuses, KP→BP daily allocations, withdrawals, payouts.
- **D-05:** Query existing tables (`BpTransaction`, `TpTransaction`, `KpEvent`) via UNION — no new table, no migration, no backfill needed.
- **D-06:** Display as a sortable list (click column headers to sort). Columns: Date, Type (bet/payout/daily bonus/KP conversion), Description (market title if bet-related), BP +/-, TP +/-.
- **D-07:** Ledger is fully public — visible on any user's profile, not just your own.

### Market Bet Details
- **D-08:** Add a participants section to market detail page showing who bet: username (linked to profile), side taken, stake amount (BP), timestamp. Full transparency — all details visible.
- **D-09:** Add aggregate stats: total BP staked, number of participants per side, average stake. Show as a summary row above the participant list.
- **D-10:** For resolved/closed markets: add a payout breakdown table showing who won, how much BP/TP each winner received.

### Claude's Discretion
- Exact placement of new sections on market detail page (above or below comments)
- Pagination strategy for participant list and ledger (cursor vs offset)
- Loading skeleton design for new sections
- How many ledger rows to show per page

</decisions>
