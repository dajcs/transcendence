<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /api/auth/oauth/{provider} | User-initiated OAuth flow; state param must be validated |
| Provider callback → /api/auth/oauth/{provider}/callback | Provider-controlled params (code, state); CSRF via state mismatch |
| Authenticated user → /api/users/data-export | Cookie auth required; must not leak other users' data |
| Authenticated user → DELETE /api/users/account | Cookie auth required; irreversible action |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing | oauth_service.handle_callback | mitigate | State validated against Redis key `oauth_state:{state}`; consumed on use (delete after read); TTL 600s |
| T-06-02 | CSRF | GET /api/auth/oauth/{provider}/callback | mitigate | `state` param acts as CSRF token; mismatch raises HTTP 400 |
| T-06-03 | Repudiation | DELETE /api/users/account | mitigate | Pseudonymization preserves audit trail (transactions, bets kept with anon_id); KP events deleted entirely per PRIVACY.md |
| T-06-04 | Information Disclosure | GET /api/users/data-export | mitigate | Route uses _get_current_user from request cookie; returns only the authenticated user's own data |
| T-06-05 | Elevation of Privilege | oauth callback redirect | mitigate | Redirect target is hardcoded to `/dashboard` (not user-supplied); no open redirect possible |
| T-06-06 | Injection | i18n t() replacement | accept | Replacements are developer-controlled keys from code, not user input fed into translation template; no XSS vector |
</threat_model>
