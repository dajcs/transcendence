## Spacing Scale

Declared values (must be multiples of 4). Uses Tailwind's default scale via utility classes — no custom tokens.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, tight inline spacing |
| sm | 8px | p-2 / gap-2 | Compact element spacing (form field label gap) |
| md | 16px | p-4 / gap-4 | Default element spacing (card padding, form gap) |
| lg | 24px | p-6 / gap-6 | Section padding (nav px, page sections) |
| xl | 32px | p-8 / gap-8 | Layout gaps (main content top padding) |
| 2xl | 48px | p-12 | Major section breaks |
| 3xl | 64px | p-16 | Page-level vertical spacing |

Exceptions:
- Touch targets for bet YES/NO buttons: minimum 44px height (use `min-h-[44px]` — accessibility requirement, not a design token)
- Comment reply indent: 24px left margin (`ml-6`) to visually distinguish parent from child

## Typography

Established by Phase 1 components (LoginForm.tsx, TopNav.tsx). Extend the same pattern.

| Role | Size | Tailwind Class | Weight | Weight Class | Line Height | Usage |
|------|------|----------------|--------|-------------|-------------|-------|
| Body | 14px | text-sm | 400 (regular) | font-normal | 1.5 | Form labels, comment text, table data, market description |
| Label | 12px | text-xs | 400 (regular) | font-normal | 1.4 | Error messages, secondary metadata (timestamp, tag) |
| Heading | 20px | text-xl | 700 (bold) | font-bold | 1.2 | Market title on detail page, section headings |
| Display | 24px | text-2xl | 700 (bold) | font-bold | 1.2 | Page-level heading (e.g., "Markets", "Dashboard") |

Rules:
- Maximum 4 sizes in use (12, 14, 20, 24). Do not introduce other sizes.
- Maximum 2 weights: 400 and 700. Do not use font-semibold (600) — keep it simple.
- Odds percentage displayed as 14px bold (`text-sm font-bold`) — exception to the 2-weight rule for a single numeric data context.

## Color

D-01 defers all custom color tokens to Phase 6. This contract uses Tailwind's built-in gray/blue/red/green palette only.

| Role | Tailwind Color | Usage |
|------|---------------|-------|
| Dominant (60%) | `bg-white` / `bg-gray-50` | Page background, form backgrounds |
| Secondary (30%) | `bg-gray-100` / `border-gray-200` | Cards, nav border, table rows, filter tab bar |
| Accent (10%) | `bg-blue-600` / `text-blue-600` | Reserved — see list below |
| Destructive | `bg-red-500` / `text-red-500` | Withdraw bet button, form error text |
| Positive signal | `text-green-600` | YES bet side label, positive odds indicator |
| Negative signal | `text-red-600` | NO bet side label |

Accent (`blue-600`) reserved for exactly these elements:
1. Primary CTA buttons: "Create Market", "Place Bet", "Post Comment"
2. Active navigation link (current page indicator)
3. Sign Up button in TopNav (existing Phase 1 pattern)
4. Sort button — active/selected state only

Accent is NOT used for: general links, info text, timestamps, table headers, secondary buttons.

Secondary buttons (non-destructive, non-primary): `border border-gray-300 rounded hover:bg-gray-50` with default text color — matching Phase 1 Logout button pattern.

## Component Patterns

Established patterns from Phase 1 code to extend consistently:

### Buttons

```
Primary:   px-4 py-2 bg-blue-600 text-white rounded hover:bg-blue-700 disabled:opacity-50 text-sm
Secondary: px-3 py-1 border border-gray-300 rounded hover:bg-gray-50 text-sm
Danger:    px-4 py-2 bg-red-500 text-white rounded hover:bg-red-600 disabled:opacity-50 text-sm
```

### Form Fields

```
Label:  block text-sm font-medium text-gray-700
Input:  mt-1 block w-full border border-gray-300 rounded px-3 py-2 text-sm
Error:  text-red-500 text-xs mt-1
```

### Cards (new for Phase 2)

```
Container: bg-white border border-gray-200 rounded p-4
```

### Filter Tabs (All / Open / Resolved)

```
Tab bar:      flex gap-2 border-b border-gray-200 mb-4
Tab default:  px-4 py-2 text-sm text-gray-600 hover:text-gray-900
Tab active:   px-4 py-2 text-sm text-blue-600 border-b-2 border-blue-600 font-bold
```

### Sort Buttons (Deadline / Active / Newest)

```
Button default:  px-3 py-1 text-sm border border-gray-300 rounded hover:bg-gray-50
Button active:   px-3 py-1 text-sm border border-blue-600 text-blue-600 rounded bg-blue-50
```

### Pagination (Previous / Next)

```
Container: flex items-center gap-4 mt-6
Button:    px-4 py-2 text-sm border border-gray-300 rounded hover:bg-gray-50 disabled:opacity-40 disabled:cursor-not-allowed
Page info: text-sm text-gray-500  (e.g., "Page 2 of 8")
```

### Comment Indent (replies)

```
Top-level comment: no left margin
Reply:             ml-6 border-l border-gray-200 pl-4
```

### Odds Display

```
Container: flex items-center gap-4
YES side:  text-sm font-bold text-green-600  (e.g., "YES 62%")
NO side:   text-sm font-bold text-red-600    (e.g., "NO 38%")
```

### Portfolio Summary Card (Dashboard)

```
Outer: grid grid-cols-3 gap-4
Cell:  bg-gray-50 border border-gray-200 rounded p-4 text-center
Label: text-xs text-gray-500 mb-1
Value: text-xl font-bold text-gray-900
```

### Balance Display (TopNav or Dashboard header)

```
Format: "bp: 12  kp: 45  tp: 3"  (space-separated, text-sm text-gray-600)
```

## Interaction States

All interactive elements must implement these states. Use Tailwind variants only.

| State | Implementation |
|-------|---------------|
| Hover | `hover:bg-*` or `hover:underline` — always present on clickable elements |
| Focus | `focus:outline-none focus:ring-2 focus:ring-blue-500` — on all inputs and buttons |
| Disabled | `disabled:opacity-50 disabled:cursor-not-allowed` — on submit buttons during loading |
| Loading | Button text changes to "[Action]ing..." (e.g., "Placing bet...") while submitting |
| Error | Red border on invalid field: `border-red-500`; error message below field |
| Active sort/tab | Blue border/text as defined in Component Patterns above |
| Upvoted | Upvote button changes to filled/bold style: `text-blue-600 font-bold` |
