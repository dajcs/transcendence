<objective>
Add unlike (remove upvote) capability for comments and markets, expose a user_has_liked flag on both CommentResponse and MarketResponse, and wire the backend to compute that flag from DB. The unlike operation deletes the upvote row and inserts an LpEvent(amount=-1) to decrement LP for the content author. This enables the Twitter-style heart toggle in the frontend (D-04, D-05, D-06).

Purpose: Frontend heart toggle requires both a like and an unlike API endpoint, plus a persisted signal (user_has_liked) to render filled/empty heart on load — for both comments AND markets (D-05).
Output: unlike_comment/unlike_market service functions, DELETE route endpoints, CommentResponse.user_has_liked field, MarketResponse.user_has_liked field, get_market wired for optional-auth user context.
</objective>

```python
async def unlike_comment(db: AsyncSession, voter_id: uuid.UUID, comment_id: uuid.UUID) -> None:
    """Remove upvote from comment; decrement LP for author by 1."""
    comment = (await db.execute(select(Comment).where(Comment.id == comment_id))).scalar_one_or_none()
    if comment is None:
        raise HTTPException(status_code=404, detail="Comment not found")
    if comment.user_id == voter_id:
        return  # self-unlike — no-op
    upvote = (await db.execute(
        select(CommentUpvote).where(
            CommentUpvote.comment_id == comment_id,
            CommentUpvote.user_id == voter_id,
        )
    )).scalar_one_or_none()
    if upvote is None:
        return  # not upvoted — no-op
    await db.delete(upvote)
    today = datetime.now(timezone.utc).date()
    db.add(LpEvent(
        user_id=comment.user_id,
        amount=-1,
        source_type="comment_upvote",
        source_id=comment_id,
        day_date=today,
    ))
    await db.commit()
```
